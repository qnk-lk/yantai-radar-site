# 抓取策略（新加坡可运行版）

## 目标
让自动化优先访问更容易成功抓取、且更有证据价值的页面，并避免区域受限来源反复失败。

## 抓取优先顺序
1. 搜索引擎定向检索具体页面
2. 具体文章页 / 公告页 / 职位页 / 案例页
3. 栏目页
4. 首页

## 抓取规则
- 不要把首页当主要抓取入口
- 如果首页抓取失败，不要反复重试
- 如果 web_fetch 失败，优先换具体页面，不要继续抓同一首页
- 如果同一来源连续两次抓取失败，切换到其他白名单来源
- 如果页面只有目录没有正文，继续向下定位详情页
- 搜索结果页和栏目页只用于导航，不作为最终证据

## 区域限制处理
如果某来源返回 403，且正文中明确提示当前地区或当前 IP 不支持访问：
- 立即停止重试该来源
- 将该来源标记为“区域受限来源”
- 不再将该来源作为高优先级自动化来源
- 切换到其他白名单来源或其他原始来源
- 在日报中如有必要，可写“该来源当前受区域限制，未纳入自动化正文证据”

## 输出规则
- 只把有正文、有时间、有主体的页面写入核心栏目
- 搜索结果页和栏目页只能作为导航，不作为最终证据
- 抓取失败时，明确写“来源访问受限”或“正文抓取失败”
- 不要把因区域限制无法访问的来源当作可验证证据
