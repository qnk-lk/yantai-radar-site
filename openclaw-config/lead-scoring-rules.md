# 销售线索评分规则

## 目标
对每条线索进行销售价值判断，而不是只做信息摘要。

## 评分维度
每条线索都尽量评估以下 5 项：

1. 需求明确度
2. 项目阶段清晰度
3. 主体明确度
4. 区域匹配度
5. 跟进可执行性

## 线索等级

### A 级线索
满足多数条件：
- 企业主体明确
- 地区明确，且在山东
- 明确提到 MES / WMS / QMS 或高度相关系统
- 有项目动作，例如招标、采购、上线、实施、验收、升级
- 能直接设计跟进动作

### B 级线索
满足部分条件：
- 企业主体明确
- 地区明确
- 有较强需求信号
- 但未完全确认项目阶段或系统模块

### C 级线索
只有早期信号：
- 招聘
- 转型表述
- 园区项目风向
- 厂商营销稿中的弱线索
需要二次核验才能进入销售名单

### D 级线索
不建议跟：
- 信息过泛
- 主体不清
- 地区不清
- 无法判断需求
- 无后续动作空间

## 常见销售信号映射

### 更像 MES 需求
- 生产调度
- 制造执行
- 车间管理
- 工单、工艺、报工、追溯
- 智能工厂、数字化车间

### 更像 WMS 需求
- 仓储管理
- 物流自动化
- 库存可视化
- 出入库流程
- 立库、仓储数字化

### 更像 QMS 需求
- 质量管理
- 质量追溯
- 检验、实验室、质量体系
- 不良分析
- 合规、过程质量控制

## 输出要求
每条线索尽量补充：
- 可能需求方向：MES / WMS / QMS / 待判断
- 线索等级：A / B / C / D
- 推荐动作：继续跟 / 观察 / 放弃
